Regression Lineaire Theorie T1

1. Definition et Types

v

L'apprentissage supervise est une methode ou le modele apprend a partir de donnees etiquetees. La regression predit une valeur continue.

Simple : y = w0 + w1*x
Multiple : y = w0 + w1*x1 + w2*x2 + ... + wp*xp
  • w0 : biais (intercept) — valeur quand x=0
  • w1, w2... : poids (coefficients) — influence de chaque feature
  • x : variables d'entree (features)
  • y : valeur cible a predire (target)

2. Fonction Cout MSE

v
J(θ) = (1/2m) * Σ(ŷᵢ - yᵢ)²

Avec m = nombre d'observations
ŷᵢ = valeur predite, yᵢ = valeur reelle
Objectif : trouver w0 et w1 qui minimisent J(θ). Plus MSE est proche de 0, meilleur est le modele.

MSE = 0 signifie que le modele predit parfaitement toutes les valeurs du dataset d'entrainement.

3. Gradient Descent

v

Algorithme iteratif qui met a jour les parametres dans la direction opposee au gradient pour minimiser le cout.

Mise a jour simultanee :
θ0 := θ0 - α * (∂J/∂θ0)
θ1 := θ1 - α * (∂J/∂θ1)

Gradients :
∂J/∂θ0 = (1/m) * Σ(ŷᵢ - yᵢ)
∂J/∂θ1 = (1/m) * Σ(ŷᵢ - yᵢ) * xᵢ
α = learning rate. Trop grand → divergence. Trop petit → convergence lente.
1

Initialisation

w0 = 0, w1 = 0

2

Prediction

Calculer ŷᵢ = w0 + w1*xᵢ pour chaque point

3

Calcul Erreur

Calculer MSE et les gradients

4

Mise a Jour

Appliquer la formule de mise a jour simultanement

5

Iteration

Repeter jusqu'a convergence (MSE minimal)

4. Implementation sklearn

v
from sklearn.linear_model import LinearRegression from sklearn.model_selection import train_test_split from sklearn.metrics import mean_squared_error # Preparation X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2) # Modele model = LinearRegression() model.fit(X_train, y_train) # Parametres print(model.intercept_) # w0 print(model.coef_) # w1, w2... # Evaluation y_pred = model.predict(X_test) mse = mean_squared_error(y_test, y_pred)
Regression Logistique T2

1. Classification Binaire

v

La regression logistique predit une probabilite entre 0 et 1 pour une classification binaire (y ∈ {0,1}).

Modele lineaire : z = w·X + b = w0 + w1x1 + w2x2...
Fonction sigmoide : P(y=1) = σ(z) = 1 / (1 + e^(-z))
Decision : y=1 si P(y=1) >= 0.5, sinon y=0
La sigmoide transforme n'importe quelle valeur reelle en probabilite entre 0 et 1.

2. Fonction Sigmoide — Valeurs cles

v

-∞≈ 0Certainement classe 0
-50.0067Tres probablement classe 0
00.5Incertain, frontiere
+50.9933Tres probablement classe 1
+∞≈ 1Certainement classe 1

3. Fonction Cout Log-Loss

v
L = -[y·log(p) + (1-y)·log(1-p)]

Si y=1 : L = -log(p) → minimiser en maximisant p
Si y=0 : L = -log(1-p) → minimiser en minimisant p

4. Gradient pour la Logistique

v
∂L/∂θ = (p - y) · x

Mise a jour : θ := θ - α · (p - y) · x
La mise a jour des poids pousse le modele dans la bonne direction. Si p > y (surpredit), on diminue θ. Si p < y (sous-predit), on augmente θ.
KNN — K-Nearest Neighbors T3

1. Principe de l'Algorithme

v

KNN est un algorithme non parametrique — il ne construit pas de modele explicite. Pour classer un nouveau point, il cherche les k points d'entrainement les plus proches et vote.

1

Calculer distances

Calculer la distance euclidienne du point P a chaque point du dataset

2

Trier

Trier tous les points par distance croissante

3

Selectionner k voisins

Prendre les k premiers points (les plus proches)

4

Vote majoritaire

La classe qui apparait le plus parmi les k voisins est la prediction

2. Distance Euclidienne

v
1D : d(A, B) = |xA - xB|

2D : d(A, B) = sqrt((x1A-x1B)² + (x2A-x2B)²)

nD : d(A, B) = sqrt(Σ(xiA - xiB)²)
KNN est sensible a l'echelle — toujours normaliser/standardiser les features avant d'utiliser KNN.

3. Choix de k

v
k petit (ex: k=1)k grand (ex: k=N)
Overfitting possibleUnderfitting possible
Tres sensible au bruitFrontieres floues
Variance eleveeBiais eleve
Regle generale : choisir k impair pour eviter les egalites. Utiliser cross-validation pour trouver le k optimal.
Arbre de Decision T4

1. Entropie

v
H(S) = -Σ pᵢ · log₂(pᵢ)

H = 0 → ensemble pur (une seule classe)
H = 1 → melange maximal (50% / 50% en binaire)

Exemple : 6 exemples, 3 OUI, 3 NON
P(OUI) = 3/6 = 0.5  →  log₂(0.5) = -1
H = -(0.5×(-1) + 0.5×(-1)) = 1

2. Gain d'Information

v
IG(S, A) = H(S) - Σ (|Sv|/|S|) · H(Sv)

Sv = sous-ensemble apres division par attribut A
On choisit l'attribut avec le plus grand gain d'information — il divise le mieux les donnees. C'est le principe de l'algorithme ID3.

3. Indice de Gini

v
Gini(S) = 1 - Σ pᵢ²

Gini = 0 → ensemble pur
Gini = 0.5 → melange maximal (binaire)

L'indice de Gini est utilise par l'algorithme CART (sklearn DecisionTreeClassifier par defaut).

4. Avantages et Limites

v
Avantages
  • Facile a interpreter
  • Pas besoin de normalisation
  • Gere variables categoriques
Limites
  • Overfitting si trop profond
  • Instable au bruit
  • Biais vers variables avec plus de valeurs
SVM — Support Vector Machine T5

1. Hyperplan Separateur

v
Hyperplan : w1·x1 + w2·x2 + b = 0

Classe +1 : w·x + b >= +1
Classe -1 : w·x + b <= -1

L'hyperplan separateur est la frontiere de decision. SVM cherche l'hyperplan qui maximise la marge entre les deux classes.

2. Marge et Vecteurs Support

v
Marge = 2 / ||w||

||w|| = sqrt(w1² + w2²)

Objectif : maximiser la marge = minimiser ||w||
Vecteurs support : points exactement sur les marges (score = ±1). Ce sont eux qui definissent l'hyperplan. Si on les retire, le modele change.

3. Pourquoi Maximiser la Marge ?

v

Une marge large signifie que le modele est robuste aux nouvelles donnees qui peuvent etre legerement differentes des donnees d'entrainement. Cela ameliore la generalisation et reduit l'overfitting.

  • TP : Vrai Positif — predit 1, reel 1

  • FP : Faux Positif — predit 1, reel 0 (Erreur type I)
  • FN : Faux Negatif — predit 0, reel 1 (Erreur type II)

2. Metriques d'Evaluation

v
Accuracy = (TP + TN) / (TP + TN + FP + FN)
Precision = TP / (TP + FP)
Recall = TP / (TP + FN)
F1-Score = 2 * (Precision * Recall) / (Precision + Recall)

3. Impact du Seuil de Decision

v

En abaissant le seuil de 0.5 a 0.3 :
→ Plus de cas sont predits positifs
→ Recall augmente (on detecte plus de vrais positifs)
→ Precision diminue (plus de faux positifs)

Exercice 3 — Questions Conceptuelles
Comprendre le Gradient Descent
  1. Que se passe-t-il si α = 10 (trop grand) ? Donner un exemple numerique.
    Les mises a jour seront trop grandes → les parametres oscillent → MSE augmente au lieu de diminuer
  2. Difference entre batch gradient descent et stochastique ?

Workshop 2 — Partie 1
Analyse du Code Gradient Descent
def learning2(X, y_real): w0 = 0 w1 = 0 nb_iterations = 50 for i in range(nb_iterations): y_predicted = w1 * X + w0 error = mse(y_predicted, y_real) print(w0, w1, y_predicted, error) w1 = w1 + 0.4 # probleme ici !
  1. Probleme 1 : w1 augmente toujours de +0.4 sans considerer la direction. Le modele peut depasser le minimum.
    Solution : Gradient Descent — utiliser la derivee pour connaitre la direction.
  2. Probleme 2 : Le pas 0.4 est fixe et peut etre trop grand ou trop petit.
    Solution : Learning rate adaptif — multiplier le gradient par α petit.
  3. Augmenter les iterations ne resout pas le probleme : si la mise a jour est mauvaise, plus d'iterations = plus d'erreur !
  4. Solution correcte : w1 = w1 - lr × (∂MSE/∂w1) en utilisant le gradient

Questions sur l'implementation
  1. Pourquoi separer train/test ?
    Evaluer la generalisation du modele sur des donnees qu'il n'a jamais vues. Sinon on mesure la memorisation, pas l'apprentissage.
